# 05 — Oppfølging (lettvektsmodell)

## Rytme
- **Check‑in (hver 6–8. uke, 20 min):** Strategisk synk. Gå gjennom hva som har dukket opp i GitHub Project pr. mål (Finne/Forstå/Løse). Identifiser tverrgående avklaringer, avhengigheter og behov.
- **Quarterly review (45 min):** Korte beslutninger/justeringer. Dokumenteres som en halv side pr. mål.

## Roller
- **Mål‑stewards (3 personer):** Én per mål (Finne/Forstå/Løse). Følger utviklingen på tvers og foreslår justeringer. Ikke operativ styring.

## GitHub‑oppsett i korte trekk
1. **Felles labels** i teamenes repoer (`pg-mål:*` og `pg-spor:*`)
2. **GitHub Project** på produktgruppenivå med tre views (Finne/Forstå/Løse)
3. **Automatisk kvartalsrapport** som oppretter `strategy/status-YYYY-QX.md`

## Mal for kvartalsnotat
Opprett en fil `strategy/status-YYYY-QX.md` og bruk malen under.
